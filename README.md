# 포장사양서 실물대조 뷰어 (Spec Cross-Check Viewer)

포장사양서(PDF)를 첨부하면 기재 항목을 구조화하고, 단상자 3D 렌더링·라벨 목업·적재 기준과 실물 사진을 한 화면에서 대조하는 QC 보조 도구입니다. 단일 HTML 파일로 동작하며 서버가 필요 없습니다.

## 주요 기능
- **00 사양서 첨부** — 텍스트형 PDF 자동 추출(pdf.js), 모든 값 클릭 편집. 품목 전환용
- **01 사양서 기준 정보** — 추출/입력값 구조화 표
- **02 단상자 3D 뷰어** — 드래그/스와이프 회전, 전면·후면·측면·밀면(착인면) 스냅, 자동회전
- **02B 실물 사진 입력** — 1안(최초 검수) / 2안(협력업체 생산) 슬롯. 가이드 프레임 촬영·크롭 후 3D 면에 실사 텍스처 적용. 밀면 적용 시 착인 기준 오버레이 표시
- **03 인/아웃박스 라벨 목업**, **04 파레트 적재 기준**
- **규칙 엔진** — 사양서 미기재 항목에 「현장 포장 사양 관리 v1 (2025-02-04)」 기준값을 `현장포장사양관리 적용 › 확인필요` 배지와 함께 자동 적용

## GitHub Pages 배포
1. GitHub에서 새 저장소 생성 (예: `spec-crosscheck`) — Private 권장(사내 도구)이나, Pages는 무료 플랜에선 Public 저장소만 지원
2. 이 폴더의 파일을 업로드 (`index.html`, `README.md`)
3. 저장소 **Settings → Pages → Source: Deploy from a branch → main / (root)** 선택 후 저장
4. 1~2분 후 `https://<계정명>.github.io/spec-crosscheck/` 접속

> https로 서빙되므로 모바일에서 **실시간 가이드 카메라**(프레임 보며 촬영)가 완전하게 동작합니다.

## 유의사항
- 첨부한 사양서·사진은 **브라우저 안에서만 처리**되며 서버로 전송되지 않습니다. 단, 새로고침 시 초기화됩니다
- pdf.js는 cdnjs CDN에서 로드됩니다 — 사내망에서 cdnjs 차단 시 PDF 자동 추출만 비활성화되고 직접 입력은 정상 동작합니다
- 사양서 원문·실물 사진 등 사내 자료를 저장소에 커밋하지 마세요 (도구 자체에는 특정 품목 데이터가 포함되지 않도록 초기값을 확인하세요)
- 3D 목업의 그래픽은 참고용 재구성이며, 공식 검수·CAPA 판정에는 현품 실사와 원본 사양서 대조가 필요합니다
