# 포장사양서 실물대조 뷰어 (Spec Cross-Check Viewer)

포장사양서(PDF)를 첨부하면 기재 항목을 구조화하고, 포장 유형별 3D 렌더링·라벨 목업·적재 기준과 실물 사진을 한 화면에서 대조하는 QC 보조 도구입니다. 단일 HTML 파일로 동작하며 서버가 필요 없습니다.

## 지원 포장 유형 (범용)
사양서의 **착인블록(용기/단상자)·컷팅유형·단품/세트** 값으로 유형을 **자동 판별**하며, 상단 카드로 수동 전환도 가능합니다. 유형에 따라 3D 모델·기준정보 항목·라벨/적재 섹션이 함께 전환됩니다.

| 유형 | 3D 모델 | 착인 기준 | 라벨/적재 섹션 |
|---|---|---|---|
| **튜브** | 원통 + 실링부(일자컷팅=직선 / R컷팅=라운드) | 실링부 압인(앞·뒤, 보통 무색) | 없음 |
| **용기/쿠션** | 원형 퍽(뚜껑·밑면) | 후면 하단 착인 | 없음 |
| **단상자** | 박스 | 밀면 착인 | 인/아웃박스 · 파레트 |
| **셋트 케이스** | 박스 + 구성품 리스트 | 밀면 착인(짧은 사용기한 품목 기준) | 인/아웃박스 · 파레트 |

## 형상 소스 우선순위 (실측 우선)
사양서에는 단상자 크기·모양이 없어 사양서만으로 그린 3D는 근사입니다. 그래서 형상은 **아래 우선순위**로 확정하고, 뷰어 우상단에 정확도 배지(`실물 반영됨`/`치수 반영`/`근사`)를 항상 표시합니다.
1. **실물 사진**(02B에서 면에 적용) → 그 위에 사양서 착인·라벨 기준 오버레이 — `실물 반영됨`
2. **형상 카드**의 프리셋 + 치수 입력(단상자·셋트 `장·폭·고`) → 3D 비율·**촬영 가이드 프레임 비율**에 반영 — `치수 반영`
3. 사양서 기반 근사 — `근사(사진 없음)`

> 형상 카드는 해당 유형의 **착인 사양이 입력되어야 활성화**됩니다(미입력 시 잠금). 튜브·용기는 현재 치수가 촬영 가이드·배지에 반영되며, 3D 치수 반영은 다음 단계입니다.

## 자재번호 자동 유형 판별
자재번호 접두로 유형을 자동 세팅합니다 — **10…=포장**(단상자/세트일 땐 셋트), **20…=충전**(컷팅유형 있으면 튜브, 없으면 용기). PDF 자동추출 시에도 이 접두를 최우선 신호로 사용합니다.

## 주요 기능
- **00 사양서 첨부** — 텍스트형 PDF 자동 추출(pdf.js) + 자재번호/착인 기반 유형 자동판별, 모든 값 클릭 편집
- **00·5 포장 유형 선택** — 4개 유형 전환 · 자동판별 결과 표시
- **형상 카드** — 프리셋·치수 입력으로 실제 비율 반영, 정확도 배지 표시
- **01·5 디자인 레퍼런스 검색·종합** — 제품명으로 네이버 블로그·쇼핑·구글이미지·쿠팡·공식몰 **검색 링크 자동 생성**, 디자인 종합 노트(편집), "종합 색상 → 3D 전면 반영"(말차그린/블랙/블루/핑크/골드 감지). ※ 브라우저 보안·판매몰 봇차단으로 페이지가 외부 사이트를 직접 긁을 수는 없어, 검색은 링크로 열고 이미지는 02B로 반영하는 방식(완전 자동 종합은 검색 API 백엔드 필요)
- **01 사양서 기준 정보** — 유형별 스키마로 렌더되는 구조화 표(튜브는 컷팅유형·완제품길이, 셋트는 구성품 등)
- **02 유형별 3D 뷰어** — 드래그/스와이프 회전, 유형별 스냅뷰(착인면 바로보기), 자동회전
- **02B 실물 사진 입력** — 1안(최초 검수) / 2안(협력업체 생산) 슬롯. 가이드 프레임 촬영·크롭 후 3D 면에 실사 텍스처 적용. 착인면 적용 시 착인 기준 오버레이 표시. 적용면 목록은 유형별로 전환(예: 용기 밑면·튜브 실링부)
- **03 인/아웃박스 라벨 목업**, **04 파레트 적재 기준** — 단상자·셋트에서만 표시
- **규칙 엔진** — 사양서 미기재 항목에 「현장 포장 사양 관리 v1 (2025-02-04)」 기준값을 `현장포장사양관리 적용 › 확인필요` 배지와 함께 자동 적용

## GitHub Pages 배포
1. GitHub에서 새 저장소 생성 (예: `spec-crosscheck`) — Private 권장(사내 도구)이나, Pages는 무료 플랜에선 Public 저장소만 지원
2. 이 폴더의 파일을 업로드 (`index.html`, `README.md`)
3. 저장소 **Settings → Pages → Source: Deploy from a branch → main / (root)** 선택 후 저장
4. 1~2분 후 `https://<계정명>.github.io/spec-crosscheck/` 접속

> https로 서빙되므로 모바일에서 **실시간 가이드 카메라**(프레임 보며 촬영)가 완전하게 동작합니다.

## 유의사항
- 첨부한 사양서·사진은 **브라우저 안에서만 처리**되며 서버로 전송되지 않습니다. 단, 새로고침 시 초기화됩니다
- pdf.js는 cdnjs CDN에서 로드됩니다 — 사내망에서 cdnjs 차단 시 PDF 자동 추출만 비활성화되고 직접 입력은 정상 동작합니다
- 사양서 원문·실물 사진 등 사내 자료를 저장소에 커밋하지 마세요 (도구 자체에는 특정 품목 데이터가 포함되지 않도록 초기값을 확인하세요)
- 3D 목업의 그래픽은 참고용 재구성이며, 공식 검수·CAPA 판정에는 현품 실사와 원본 사양서 대조가 필요합니다
